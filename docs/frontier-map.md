# 具身智能前沿方向地图

## 1. VLN：大范围语言导航

VLN 更接近“我要去哪里、按什么路线去、路上如何纠错”。常见抽象是把机器人视为质点或移动平台，重点在空间理解、地图记忆、长程规划和语言 grounding。

| 细分方向 | 关注问题 | 典型本体 | 判断标准 |
| --- | --- | --- | --- |
| 室内 VLN | 按自然语言在建筑内导航 | 轮式、四足、人形 | 是否能在连续环境中稳定闭环，而不是只在离散 nav graph 上好看 |
| 户外/街景导航 | 长距离目标定位和路径规划 | 无人车、移动机器人 | 是否能处理大尺度地图、遮挡、动态环境 |
| 空中导航 | 三维空间中的语言导航 | 无人机 | 是否能把语言目标转成安全航迹 |
| 具身交互导航 | 边问边走、边看边修正 | 服务机器人、人形 | 是否具备错误发现和恢复能力 |

VLN 的趋势不是单纯更大的语言模型，而是语言、地图、记忆、约束规划之间的组合。

## 2. VLA / WAM：从动作生成到世界动作模型

VLA 关注“看到什么、听到什么、下一步怎么动”。在真实机器人上，主战场通常是机械臂操作，继续细分到双臂、灵巧手、触觉和接触丰富任务。

| 细分方向 | 关注问题 | 典型本体 | 判断标准 |
| --- | --- | --- | --- |
| Generalist VLA | 一个模型覆盖多个操作任务 | 单臂、双臂 | 跨任务、跨场景、跨机器人是否真实成立 |
| Diffusion / Flow Policy | 连续动作生成 | 机械臂 | 是否稳定、低延迟、对分布外扰动可恢复 |
| Dexterous Manipulation | 高自由度灵巧手 | 灵巧手、双臂灵巧手 | 是否处理接触、滑移、力控和物体多样性 |
| WAM | 同时建模未来世界状态和动作 | 机械臂为主 | 是否比纯 VLA 更能规划、验证和纠错 |

VLA 的风险是把“会说”和“会动”绑定得太死。WAM 的价值在于引入可预测的未来状态，让策略不只是直接吐动作。

## 3. Agentic Planning：大脑规划，小脑执行

一个更值得跟踪的趋势是：不一定把所有能力塞进单个 VLA 大模型，而是让 agent 负责分解任务、选择技能、调用工具、检查失败；底层由小模型、传统控制器或技能库执行。

| 层级 | 作用 | 可用方法 |
| --- | --- | --- |
| 任务 agent | 把用户目标拆成子任务 | LLM/VLM planner、行为树、PDDL、程序生成 |
| 技能选择 | 选择可执行技能或控制接口 | affordance scoring、skill library、retrieval |
| 执行控制 | 低延迟闭环动作 | classical control、MPC、diffusion policy、小 VLA |
| 监控恢复 | 发现失败并重规划 | verifier、world model、视觉/触觉反馈 |

这里的核心判断：泛化性可能来自清晰的层级分工，而不是更大的端到端动作模型。

## 4. Lightweight Autonomy：机器人本地自决策

单个机器人长期运行时，需要在弱网、断网、低算力和安全约束下自决策。轻量化不是只压模型参数，而是把系统拆成“本地必须闭环”和“云端可增强”。

| 方向 | 要解决的问题 |
| --- | --- |
| 端侧小模型 | 低延迟理解、状态判断、技能选择 |
| 动作先验压缩 | 把大 VLA/WAM 蒸馏成可部署策略 |
| 层级缓存与记忆 | 避免每次都重新推理完整历史 |
| 安全控制回路 | 遇到碰撞、越界、力异常时本地兜底 |
| 多机器人/边缘协同 | 单机自治为主，边缘节点做任务分配和重计算 |

调研时优先记录是否有真实机器人、实时指标、端侧算力说明和失败案例。
