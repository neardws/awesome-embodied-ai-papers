# 2026-06-28 具身智能前沿调研

## 结论

当前最值得押注的方向不是“再做一个更大的端到端 VLA”，而是层级式具身智能系统：

```text
任务理解 / 细粒度规划 agent
        ↓
世界状态预测 / 可执行性验证
        ↓
技能库、小模型策略、传统控制器
        ↓
本地安全闭环和低延迟执行
```

这个判断来自四个现象：

1. VLN 的痛点从语言理解转向真实机器人上的连续导航、跨本体泛化和失败恢复。
2. VLA 已经证明大模型知识能迁移到机器人动作，但数据、算力、实时性和跨机器人泛化仍是硬约束。
3. WAM 把“预测未来状态”和“生成动作”放在一起，适合作为规划/验证层，而不只是替代控制器。
4. 轻量化 VLA、小模型策略和 agentic planning 同时升温，说明社区在寻找比单体大模型更可部署的路线。

## 方向 1：VLN 从“导航 benchmark”走向“物理本体”

VLN 仍然是大范围规划最清晰的入口。经典任务如 [R2R](https://arxiv.org/abs/1711.07280) 和 [VLN-CE](https://arxiv.org/abs/2004.02857) 建立了语言导航评价，但主要还是室内环境和相对简化动作空间。

近期更值得关注的是两个变化：

- **跨本体导航**：[NavFoM](https://openreview.net/forum?id=QNDX2YxMa6) 关注 humanoid、quadruped、drone、autonomous vehicle 等不同本体的导航基础模型，贴近“同一规划能力迁移到不同机器人”的问题。
- **真实物理导航**：[VLN-PE](https://arxiv.org/abs/2507.13019) 明确把 VLN 放到 physical embodied environments 中，强调从仿真/离散图走向真实机器人闭环。

判断：VLN 不应只看成功率，还要看连续控制、真实传感、动态障碍、安全约束、跨本体迁移。大范围规划可以把本体抽象成质点，但最终必须知道“质点下面是什么机器人”。

## 方向 2：VLA 是主战场，但单体大模型不是唯一答案

VLA 的代表路线包括 [RT-2](https://arxiv.org/abs/2307.15818)、[OpenVLA](https://arxiv.org/abs/2406.09246)、[Octo](https://arxiv.org/abs/2405.12213)、[pi0](https://arxiv.org/abs/2410.24164)。这些工作证明视觉-语言知识可以转成机器人动作，主战场是机械臂操作，进一步延伸到双臂和灵巧手。

但 VLA 的主要约束也很明确：

- 数据昂贵：真实机器人操作数据远比 web 图文数据少。
- 实时性紧：动作生成不能像聊天模型一样慢。
- 跨本体难：不同机械臂、夹爪、灵巧手的动作空间差异很大。
- 可解释性弱：失败时很难判断是 perception、planning、policy 还是 control 出错。

判断：VLA 仍要跟，但更适合作为“技能执行器”或“动作先验”，不宜默认承担完整任务规划。

## 方向 3：WAM 是 VLA 之后的重要桥接层

[World Action Models](https://arxiv.org/abs/2605.12090) 把 WAM 定义为同时建模未来世界状态和动作生成的范式。它和纯 VLA 的差异在于：不是直接从图像/语言吐动作，而是显式或隐式预测“执行后世界会怎样”。

值得关注的代表线索：

- [Video Language Planning](https://arxiv.org/abs/2310.10625)：用视频-语言生成做长程规划。
- [Latent Action Pretraining](https://arxiv.org/abs/2410.11758)：从视频中学习 latent action，缓解机器人动作数据不足。
- [Video Prediction Policy](https://arxiv.org/abs/2412.14803)：用预测式视觉表征做通用机器人策略。

判断：WAM 最适合放在 agent 和低层控制之间，做想象、验证、动作先验或失败恢复；不一定要变成一个覆盖所有环节的超大模型。

## 方向 4：Agentic planning 是近期更可落地的系统路线

从 [SayCan](https://arxiv.org/abs/2204.01691)、[Code as Policies](https://arxiv.org/abs/2209.07753)、[VoxPoser](https://voxposer.github.io/) 到 [ReKep](https://rekep-robot.github.io/)，一个清晰趋势是让大模型负责语义规划、约束生成、技能组合和重规划，让底层控制器或小策略执行。

这条路线的优势：

- 容易复用传统控制、MPC、运动规划和已有技能库。
- 更容易调试：规划错、感知错、控制错可以分层定位。
- 数据需求低于端到端大 VLA。
- 更符合未来单机器人本地自决策：高层可慢，底层必须快。

风险也存在：agent 规划容易产生不可执行步骤，所以必须有 affordance、约束检查、世界模型或执行监控。

判断：如果做一个面向真实机器人的研究路线，优先做“agent 细粒度规划 + 小模型/传统方法执行 + 本地安全回路”，比直接训练大 VLA 更现实。

## 方向 5：轻量化不是锦上添花，而是部署前提

机器人端侧运行会遇到功耗、延迟、断网、安全兜底等约束。轻量化方向包括：

- 小型 VLA：[SmolVLA](https://huggingface.co/blog/smolvla)、[TinyVLA](https://arxiv.org/abs/2409.12514)
- 高效动作表示：[FAST](https://www.physicalintelligence.company/research/fast)
- 本地机器人模型：[Gemini Robotics On-Device](https://deepmind.google/models/gemini-robotics/gemini-robotics-on-device/)

判断：轻量化应该和系统分层一起看。大模型可以做离线蒸馏、技能生成和云端增强；机器人本体上必须保留低延迟感知、局部规划、执行监控和安全停止。

## 建议跟踪顺序

1. **先跟 VLN 物理化和跨本体**：NavFoM、VLN-PE、连续环境 VLN。
2. **再跟 agentic planning**：SayCan、VoxPoser、ReKep、Code-as-Monitor、AO-Planner。
3. **并行跟 WAM**：重点看是否能做规划验证和失败恢复，而不是只看视频生成质量。
4. **保留 VLA 基线**：OpenVLA、Octo、pi0、RT-2，用作动作模型 baseline。
5. **每轮都检查轻量化**：是否有端侧模型、延迟、算力、真实机器人测试。

## 可以形成的研究题目

### 题目 A：面向跨本体 VLN 的 agentic navigation

目标：同一语言任务规划器适配人形、四足、无人机、无人车等本体，底层由不同控制器执行。

关键点：

- 高层统一任务表示。
- 中层根据本体生成可执行约束。
- 低层复用本体控制器。
- 用真实/半真实环境验证连续导航。

### 题目 B：面向机械臂操作的 agent + WAM 验证

目标：agent 拆解操作任务，WAM 预测候选动作后果，小策略或传统方法执行。

关键点：

- 任务分解和技能检索。
- WAM 做动作后果验证。
- 执行失败后重规划。
- 与端到端 VLA 做泛化性对比。

### 题目 C：轻量化自主机器人决策栈

目标：把大模型能力压到端侧可用的层级系统，而不是直接部署一个大 VLA。

关键点：

- 小模型做状态识别和技能选择。
- 传统控制保证实时和安全。
- 云端只做慢速增强。
- 记录延迟、功耗、断网表现和失败恢复。

## 当前判断

如果只选一个方向，优先选 **agentic planning + WAM/小策略执行**。理由是它同时覆盖用户关心的几个点：VLN 的大范围规划、VLA/WAM 的动作层、轻量化部署，以及未来单机器人自我决策。

单体 VLA 仍值得作为 baseline 和动作先验跟踪，但不建议作为唯一主线。
